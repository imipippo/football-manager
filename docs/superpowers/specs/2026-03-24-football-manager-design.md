## 1. 产品概述

### 1.1 产品定位

- **类型**：足球俱乐部经营模拟手游
- **对标**：Football Manager（深度经营），专为手机碎片化时间设计
- **差异化**：战术沙盒 + 地区生态 + 球员培养

### 1.2 核心Slogan

每座城市，都值得一支冠军

### 1.3 目标用户

- 足球爱好者
- 经营类游戏玩家
- 碎片时间较多的玩家

## 2. 核心玩法

### 2.1 三大支柱

1. **战术沙盒**
   - 足球的不确定性来自阵型、打法、球员特性之间的化学反应
   - 不完全是数值游戏，战术是基础，球员状态是放大器
   - 赛前战术设置 + 赛中干预

2. **地区生态**
   - 以市为单位，不同地区玩家数量和质量影响该地区
   - 地区之间定期组织官方比赛（世界杯）

3. **球员培养与转会**
   - 足球核心是球员
   - 球员培养和交易是球队运营的核心

### 2.2 玩家爽点

- 比赛的胜利
- 获得更强的明星球员
- 阵容有了提升

### 2.3 玩家登录动力

**核心动力 = 球队实力增长 + 球队战绩**

## 3. 设计决策

### 3.1 地区生态平衡

**问题**：强地区会越来越强，弱地区越来越弱

**解决方案**：
- 弱地区 = 竞争小 + 容易升级 → 快速获得高级联赛资源
- 强地区 = 竞争大 + 难升级 → 即使资源多，也要争夺
- 天才爆率统一 → 给弱地区希望

**两种发展路径**：
1. 弱地区路线：低竞争 → 快速升级 → 获得高级联赛资源
2. 强地区路线：高竞争 → 激烈对抗 → 获得地区繁荣度加成

### 3.2 NPC填充机制

**问题**：NPC球队如何设置强度？

**解决方案**：
- NPC作为"守门员"，主要在市级联赛
- 强度适中，给新玩家过渡
- 确保玩家不会轻易降级
- 随着玩家数量增长，NPC逐渐被真实玩家替代

### 3.3 比赛引擎随机性

**问题**：比赛结果的可预测性和公平性如何平衡？

**解决方案**：中等随机性

| 因素 | 占比 | 作用 |
|------|------|------|
| 球员实力 | 50-60% | 核心因素，强队有优势 |
| 战术执行 | 20-30% | 策略深度，战术相克增加变数 |
| 随机因素 | 15-25% | 不确定性，弱队有机会爆冷 |

### 3.4 付费平衡

**问题**：如何确保非付费玩家通过时间追赶付费玩家？

**解决方案**：A+B+C综合方案

| 策略 | 具体实现 | 效果 |
|------|----------|------|
| 时间追赶 | 增加免费训练获取途径 | 非付费玩家通过时间投入可追赶 |
| 上限一致 | 属性上限、潜力上限一致 | 付费只是加速，不是变强 |
| 差异化体验 | 付费=便利；非付费=挑战 | 两种玩家都有各自乐趣 |

### 3.5 社交系统优先级

**问题**：社交系统在MVP中的优先级？

**解决方案**：简化版本

- ✅ 基础好友列表（添加、删除、查看状态）
- ✅ 系统消息替代私信（转会通知、比赛邀请等）
- ❌ 一对一私信 → 后期扩展
- ❌ 地区聊天 → 后期扩展

### 3.6 新手留存

**问题**：新手留存最关键的设计？

**解决方案**：核心动力是球队实力增长+战绩

## 4. 比赛引擎设计

### 4.1 架构

**事件驱动 + 状态机模型**

| 模型 | 作用 | 好处 |
|------|------|------|
| 状态机 | 管理比赛流程 | 结构清晰，逻辑可控 |
| 事件驱动 | 生成具体事件 | 丰富多变，真实感强 |

### 4.2 状态机层

```
比赛状态：开球、进行中、暂停、结束
球权状态：主队控球、客队控球、争球
场地区域：后场、中场、前场、禁区
时间节点：每15分钟为一个阶段
```

### 4.3 事件驱动层

```
基础事件：传球、盘带、射门、犯规、角球、任意球
关键事件：进球、点球、红黄牌、换人
特殊事件：伤病、乌龙、绝杀
事件链：传球→盘带→射门→进球
```

### 4.4 战术相克影响

| 战术 | 克制效果 | 事件概率变化 |
|------|----------|--------------|
| 高位压迫 vs 快速反击 | 反击方射门+15% | 压迫方体力消耗快 |
| 快速反击 vs 控球打法 | 反击方射门+20% | 控球方传球成功率-10% |
| 控球打法 vs 边路进攻 | 控球方控球率+10% | 边路方传中+15% |

### 4.5 心理变量动态系统

**变化因素**：

1. **事件触发**
   - 进球：全队信心+10%，进球者信心+20%
   - 失球：全队信心-10%，防守者信心-15%
   - 犯规：犯规者专注-5%，受害者信心+5%

2. **时间衰减**
   - 60分钟后：全队专注-5%/10分钟
   - 75分钟后：意志力决定是否坚持
   - 90分钟后：绝杀阶段，意志力+20%效果

3. **比分影响**
   - 领先1球：信心+5%，专注-5%（轻敌）
   - 领先2球+：信心+10%，专注-10%（松懈）
   - 落后1球：抗压决定信心变化
   - 落后2球+：触发"绝境状态"

4. **绝境逢生**
   - 触发条件：落后2球+ 且 时间>75分钟
   - 效果：意志力高的球员进入"逆转状态"
   - 概率：意志力 × 0.5%

**心理素质能力影响**：

| 心理属性 | 影响内容 |
|----------|----------|
| 抗压能力 | 落后时信心下降幅度 = 基础下降 × (1 - 抗压/100) |
| 意志力 | 绝境状态触发概率 = 意志力 × 0.5% |
| 专注力 | 时间衰减幅度 = 基础衰减 × (1 - 专注/100) |
| 自信心 | 事件触发后信心变化幅度 = 基础变化 × (1 + 自信/200) |
| 领导力 | 队长在场时，全队心理变化幅度减半 |

### 4.6 赛中干预机制

**即时生效 + 条件加成**

| 干预类型 | 基础效果 | 条件加成 |
|----------|----------|----------|
| 换人 | 新球员立即上场 | 落后时换前锋：射门+15% |
| 战术调整 | 立即改变事件概率 | 绝境时调整：意志力高效果+20% |
| 提示指令 | 立即影响球员行为 | 无 |

## 6. 技术方案

### 6.1 技术栈

| 层级 | 技术选型 | 原因 |
|------|----------|------|
| 客户端 | React Native | 跨平台，一套代码iOS+Android |
| 后端 | Node.js + TypeScript | 开发效率高，适合实时通信 |
| 主数据库 | MongoDB | 灵活的数据结构，适合游戏数据 |
| 缓存 | Redis | 会话、排行榜、实时数据 |

### 6.2 开发周期

| 阶段 | 内容 | 时间 |
|------|------|------|
| 第一阶段 | 核心系统（账号、球员、比赛、训练） | 2-3个月 |
| 第二阶段 | 经济系统、转会系统、联赛系统 | 1-2个月 |
| 第三阶段 | 青训系统、教练系统、社交系统 | 1-2个月 |
| 第四阶段 | 测试、优化、上线 | 1个月 |
| **总计** | MVP上线 | **5-8个月** |
