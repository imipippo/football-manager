# 足球俱乐部经营游戏 PRD

## 1. 产品概述

### 1.1 产品定位
- **类型**：足球俱乐部经营模拟手游
- **对标**：Football Manager（深度经营），但专为手机碎片化时间设计
- **差异化**：战术沙盒 + 地区生态 + 球员培养

### 1.2 核心Slogan
每座城市，都值得一支冠军

### 1.3 目标用户
- 足球爱好者
- 经营类游戏玩家
- 碎片时间较多的玩家

---

## 2. 核心玩法

### 2.1 三大支柱

1. **战术沙盒**
   - 足球的不确定性来自阵型、打法、球员特性之间的化学反应
   - 不完全是数值游戏，战术是基础，球员状态是放大器
   - 赛前战术设置 + 赛中干预

2. **地区生态**
   - 以市为单位，不同地区玩家数量和质量影响该地区：
     - 球队价值
     - 球迷数量
     - 青训球员质量
   - 地区之间定期组织官方比赛（世界杯）

3. **球员培养与转会**
   - 足球核心是球员
   - 球员培养和交易是球队运营的核心

### 2.2 玩家爽点
- 比赛的胜利
- 获得更强的明星球员
- 阵容有了提升

## 3. 地区与联赛体系

### 3.1 行政区划

| 级别 | 联赛 | 说明 |
|------|------|------|
| 1 | 市级C级 | 20队，多个联赛 |
| 2 | 市级B级 | 20队，多个联赛 |
| 3 | 市级A级 | 20队，多个联赛 |
| 4 | 省级B级 | 每省1个，20队 |
| 5 | 省级A级 | 每省1个，20队 |
| 6 | 全国B级 | 2个联赛，各20队 |
| 7 | 全国A级 | 1个联赛，20队 |

- 玩家不足20队时用NPC补充
- 玩家数量增长时可扩展联赛数量

### 3.2 地区限制与匹配

**地区限制：**
- 球队只进入其所在地区的联赛体系
- 例如：武汉球队 → 只进入湖北市级联赛和省级联赛
- 上海球队 → 只进入上海市级联赛和省级联赛

**NPC填充：**
- 联赛20队，用户不足时用NPC填充
- NPC球队作为固定对手存在

**地区更换规则：**
- 赛季中：不允许更换地区
- 赛季结束后：可以更换到其他地区
- 更换地区后进入新地区对应等级的联赛

### 3.3 升降级规则

- **升级**：每级前3名有升级机会
  - 上级联赛空位 ≥ 资格球队数 → 直接晋级
  - 空位 < 资格球队数 → 附加赛
- **降级**：每级最后3名降级
- **升降级后分配**：升降级球队进入哪个具体联赛完全随机，不保证进入原来所在的联赛编号

### 3.4 洲际杯赛

- **5大洲**：亚洲、欧洲、非洲、南美洲、北美洲
- **每洲3级**：冠军杯、联盟杯、挑战杯
- **资格分配**：联赛名次 + 洲际积分

### 3.5 国家队/世界杯

- 每2个赛季举办一次
- 国家冠军队玩家担任Manager
- Manager挑选球员组成国家队
- 球员镜像参赛，不影响原俱乐部

## 4. 比赛系统

### 4.1 呈现方式
- 初期：文字直播
- 后续可升级：2D画面 → 3D画面

### 4.2 比赛模拟

**赛中干预（玩家可操作）：**
| 干预类型 | 时机 |
|----------|------|
| 换人 | 比赛中途 |
| 战术调整 | 比赛中途 |
| 提示指令 | 比赛中途 |
| 比赛道具 | 只能赛前使用 |

**关键时刻：**
- 点球、任意球、角球等 → 自动处理

### 4.3 比赛引擎逻辑

- 战术相克体系（类似剪刀石头布扩展版）
- 动态事件链（战术+球员状态决定事件概率）
- 球员变量（决策、信心、注意力、意志力）是核心不确定性

### 4.4 战术系统

**阵型选择：**

| 阵型 | 说明 |
|------|------|
| 4-3-3 | 经典进攻阵型，4后卫+3中场+3前锋 |
| 4-4-2 | 均衡阵型，攻守平衡 |
| 3-5-2 | 中场强势，翼卫插上进攻 |
| 4-2-3-1 | 防守反击常用，单前锋 |
| 5-3-2 | 防守阵型，3中卫+2翼卫 |
| 4-1-4-1 | 中场密集，防守稳健 |

**战术风格：**

| 风格 | 说明 |
|------|------|
| 高位压迫 | 前场逼抢，体力消耗大 |
| 快速反击 | 防守为主，快速通过中场 |
| 控球打法 | 保持球权，传球为主 |
| 边路进攻 | 主要从边路发起进攻 |
| 中路渗透 | 短传配合，渗透中路 |

**战术相克关系：**

| 我的战术 | 克制 | 被克制 |
|----------|-------|--------|
| 高位压迫 | 快速反击 | 控球打法 |
| 快速反击 | 控球打法 | 高位压迫 |
| 控球打法 | 边路进攻 | 快速反击 |
| 边路进攻 | 高位压迫 | 中路渗透 |
| 中路渗透 | 控球打法 | 边路进攻 |

> 相克关系形成循环克制关系。

## 5. 球员系统

### 5.1 属性框架

| 模块 | 属性 |
|------|------|
| **身体** | 速度、耐力、力量、灵活性，体质（受伤概率、恢复速度，体力恢复速度） |
| **技术** | 传球、射门、盘带、停球、定位球、拦截、抢断、侵略性、盯人、头球 |
| **心理** | 自信心、抗压能力、专注力、意志力、领导力、团队协作 |
| **足球智商** | 对外单一属性，对内影响：战术执行度、场上判断、无球跑动、对手阅读、位置适应速度、创造力 |

**其他属性：**
- 左右脚熟练度：训练积累，天赋决定非惯用脚上限
- 位置熟练度：动态值，上场时间积累，足球智商决定积累速度

### 5.2 球员成长

- **潜力值**：隐性，玩家看不到具体数值
- **潜力提示**：通过球探报告、训练反馈、比赛解说等方式呈现
- **成长曲线类型**：早熟型（22-24岁巅峰）、标准型（25-27岁巅峰）、大器晚成型（28-30岁巅峰），隐性
- **属性上限**：天赋决定，身体和技术都可通过训练提升但有天花板

### 5.3 生命周期

| 维度 | 规则 |
|------|------|
| 身体 | 青训 → 成长 → 巅峰 → 衰退 → 退役，30岁后逐年下降 |
| 技术 | 相对身体滞后，衰退更慢 |
| 心理（领导力、团队协作） | 与技术能力同步成长，顶峰后不下降 |

### 5.4 退役与转型

- 退役后可转型：教练、青训教练
- 转型条件：满足一定条件（职业生涯长度、领导力、足球智商等）
- 教练能力来源：球员时期属性加权组合

## 6. 教练系统

### 6.1 教练团队组成

| 岗位 | 职能 | 影响 |
|------|------|------|
| 主教练 | 整体战术、比赛指挥 | 战术执行效果 |
| 守门员教练 | 门将训练 | 门将专项属性 |
| 体能教练 | 康复、力量、耐力 | 身体属性训练效果 |
| 防守教练 | 后防线训练 | 防守相关技术属性 |
| 心理教练 | 心理训练 | 心理属性训练效果 |
| 技术分析师 | 数据分析、对手分析 | 比赛报告、对手情报 |
| 进攻/战术教练 | 进攻套路、战术演练 | 进攻相关技术属性 |
| 助理教练 | 辅助主教练、日常训练 | 整体训练效果加成 |
| 青训教练 | 青年球员培养 | 青训产出质量 |

### 6.2 教练流动性

- **转型**：球员退役后随机选择是否转型为教练，尊重球员意愿
- **离开**：只有两种情况会离开：
  - 玩家主动解约
  - 到年龄自然退休
- **挖角**：其他俱乐部可发起挖角请求，**决定权在玩家手里**
  - 玩家选择放人 → 获得补偿金
  - 玩家选择留下 → 教练继续留队
- **教练不会主动离开**，不存在因成绩差或薪资低而自行出走的情况

## 7. 青训系统

### 7.1 梯队结构

| 梯队 | 人数 | 补充机制 | 球员年龄 |
|------|------|----------|----------|

| 一线队 | 18-35人 | 转会市场、U20上调 | 15岁+ |

### 7.2 青训球员来源

| 来源 | 说明 |
|------|------|
| 俱乐部青训中心 | 每赛季自动补充，直接归属俱乐部 |
| 转会市场 | 系统投放，玩家可见可购买 |

- 其他玩家可见：大致情况（位置、大致潜力区间）
- 详细数据不可见：具体属性、心理特质、成长曲线类型

### 7.4 地区青训质量

- 地区整体实力影响青训球员初始属性概率分布

- 弱地区：低初始属性为主，但偶尔有天才（随机爆冷）
- 成长质量看球队经营

## 8. 经济系统

### 8.1 货币体系

| 货币 | 获取 | 消耗 |
|------|------|------|
| **基础货币** | 经营收益（比赛奖金、球票、赞助、转播、周边、转会） | 工资、球场维护、球探、青训、运营成本 |
| **游戏币** | 充值、日常活动 | 道具、兑换基础货币、组织比赛奖金 |

- **兑换关系**：单向兑换（游戏币 → 基础货币）

### 8.2 核心原则

**高竞争力 = 高支出**
- 强队运营成本远超收入
- 收入增长追不上支出增长

**动态起伏**
- 资金充足但竞争力下降
- 竞争力强但资金紧张
- 两者难以同时保持

### 8.3 收入来源

| 来源 | 强队占比 | 弱队占比 |
|------|----------|----------|
| 比赛奖金 | 高 | 低 |
| 转播分成 | 高 | 低 |
| 赞助 | 高 | 中 |
| 球票 | 中 | 中 |
| 周边销售 | 低 | 低 |
| 转会收入 | 波动大 | 波动大 |

### 8.4 支出结构

| 来源 | 说明 |
|------|------|
| 球员工资 | 主要支出，强队极高 |
| 教练工资 | 视教练团队规模 |
| 青训运营 | 可变 |
| 球场维护 | 固定 |
| 球探费用 | 可变 |
| 转会抽成 | 交易金额20% |

## 11. 杯赛体系

### 11.1 联赛杯

**参赛球队：**
- 升班马（上赛季升级的3支球队）
- 上赛季联赛13-17名（共5队）
- 共8队

**赛制：**
- 第一轮：8队单场淘汰（4场）
- 16强：联赛另外12队加入，两回合淘汰
- 决赛：单场定胜负
- 共19场

### 11.2 中国杯

**参赛球队：**
- 国家A级联赛球队直接进入32强
- 其余球队通过预选赛争夺12个32强名额

**赛制：**
- 预选赛：单轮淘汰赛
- 32强：两回合淘汰赛
- 决赛：单场定胜负
- 共31场

## 12. 世界杯与洲际杯赛

### 12.1 世界杯

**基本信息：**
- 举办频率：每 2 个赛季举办一次
- 参赛球队：每个国家 1 支代表队
- Manager：该国家上赛季联赛冠军玩家担任

**国家队选拔：**
- 选拔权限：可选择任何该国籍球员
- 球员搜索：通过搜索功能寻找球员
- 队伍规模：23名队员
- 球队限制：每个俱乐部最多选择5名球员

**赛制：**
- 小组赛：8 个小组，每组 4 队
- 淘汰赛：16 强 → 8 强 → 4 强 → 决赛
- 小组赛单循环，淘汰赛单场决胜

**参赛资格：**
| 资格 | 球队来源 |
|------|----------|
| 32强 | 各国联赛冠军自动入围 |
| 预选赛 | 各国联赛非冠军球队争夺剩余名额 |

**比赛特性：**
- 球员参赛方式：镜像参赛，不影响原俱乐部
- 比赛影响：不影响原俱乐部的比赛和球员状态

**奖励分配：**
| 奖励对象 | 说明 |
|----------|------|
| 俱乐部 | 球员征用的奖励直接发给该球员所在俱乐部 |
| 国家用户 | 世界杯名次奖励给到该国家所有用户 |

**世界杯奖励：**
| 名次 | 奖励 |
|------|------|
| 冠军 | 大量游戏币 + 专属称号 + 繁荣度大幅提升 |
| 亚军 | 中量游戏币 + 称号 + 繁荣度提升 |
| 4强 | 少量游戏币 + 称号 + 繁荣度小幅提升 |

### 12.2 洲际杯赛

**基本信息：**
- 5 大洲：亚洲、欧洲、非洲、南美洲、北美洲
- 每洲 3 级：冠军杯、联盟杯、挑战杯

**赛制：**
- 冠军杯：各洲最强球队参赛，32 强
- 联盟杯：次级球队参赛，32 强
- 挑战杯：第三级别球队参赛，32 强

**资格分配：**
| 杯赛 | 资格来源 |
|------|----------|
| 冠军杯 | 各国联赛冠军 + 洲际积分排名 |
| 联盟杯 | 各国联赛 2-4 名 |
| 挑战杯 | 各国联赛 5-8 名 |

**洲际积分：**
- 球队参加洲际杯赛的成绩计入洲际积分
- 洲际积分排名影响下届杯赛资格

**奖励：**
| 名次 | 奖励 |
|------|------|
| 冠军 | 大量游戏币 + 大量球迷 |
| 亚军 | 中量游戏币 |
| 4强 | 少量游戏币 |

## 13. 足球繁荣度系统

### 13.1 两级繁荣度

**地区级繁荣度：**
- 定义：衡量一个城市/地区足球发展水平的指标
- 影响：本地区青训球员质量、本地区球迷增长、本地区赞助金额
- 影响因素：该地区俱乐部的联赛成绩、该地区活跃玩家数量

**国家级繁荣度：**
- 定义：衡量一个国家足球发展水平的指标
- 影响：全国青训球员质量、全国球迷增长、洲际杯/世界杯成绩
- 影响因素：各地区繁荣度汇总、洲际杯成绩、世界杯成绩

### 13.2 繁荣度计算

**地区繁荣度计算：**
```
地区繁荣度 = 基础值 + 本地球队成绩 + 本地玩家活跃度 + 国家繁荣度增益
```

**国家级繁荣度计算：**
```
国家级繁荣度 = 各地区繁荣度平均值 + 洲际加成 + 世界杯加成
```

### 13.3 繁荣度特性

- **动态且相对**：繁荣度不是固定值，而是相对于其他国家/地区
- 如逆水行舟，不进则退
- 你发展得好，繁荣度上升；其他玩家发展得更好，你的繁荣度相对下降

### 13.4 繁荣度调整规则

每个赛季结束后，计算每个国家/地区的综合表现得分，调整繁荣度：

**综合得分计算：**
```
综合得分 = 联赛得分 + 洲际杯得分 + 世界杯得分 + 活跃度得分
```

| 得分项 | 计算方式 |
|--------|----------|
| 联赛得分 | 该国/地区所有球队平均排名 |
| 洲际杯得分 | 该国球队在洲际杯的最佳成绩 |
| 世界杯得分 | 该国在世界杯的成绩 |
| 活跃度得分 | 该国/地区活跃玩家数量 |

**繁荣度变化：**
| 排名 | 繁荣度变化 |
|------|------------|
| 前 1/3 | 繁荣度上升（+3~+5） |

### 17.5 薪资计算

**基础公式：**
```
周薪 = 综合评分 × 基准系数 × 年龄系数
```

**基准系数：**
| 综合评分 | 基准系数 |
|----------|----------|
| 50-65 | 0.8 |
| 66-78 | 1.2 |
| 79-88 | 2.0 |
| 89-95 | 3.5 |
| 96-100 | 6.0 |

**年龄系数：**
| 年龄 | 系数 |
|------|------|
| 15-20 | 0.5 |
| 21-26 | 1.0 |
| 27-30 | 1.1 |
| 31-33 | 0.85 |
| 34+ | 0.6 |

**动态预期区间：**
```
最低接受 = 基础预期 × 0.7
最高预期 = 基础预期 × 1.5
```

| 调整因素 | 影响 |
|----------|------|
| 本赛季平均评分 > 7.5 | 预期上浮10% |
| 本赛季平均评分 < 6.0 | 预期下浮10% |
| 有竞争报价 | 预期上浮20% |
| 球员忠诚度高 | 预期下浮15% |
| 合同剩余 < 1年 | 预期下浮20% |
| 球员野心高 | 预期上浮15% |

### 17.6 转会费计算

**基础公式：**
```
转会费 = 周薪 × 52 × 合同剩余年限 × 潜力系数
```

**潜力系数：**
| 潜力等级 | 系数 |
|----------|------|
| 普通 | 1.0 |
| 较高 | 1.5 |
| 高 | 2.5 |
| 极高 | 4.0 |

**动态预期区间：**
```
最低接受价 = 基础转会费 × 0.6
最高预期价 = 基础转会费 × 2.0
```

| 调整因素 | 影响 |
|----------|------|
| 本赛季平均评分 > 7.5 | 预期上浮10% |
| 本赛季平均评分 < 6.0 | 预期下浮10% |
| 多家俱乐部竞争 | 预期上浮30% |
| 球员主动要求转会 | 预期下浮25% |
| 卖方财务紧张 | 预期下浮20% |
| 球员不想离开 | 谈判可能失败 |

### 17.7 转会市场规则

- 只有主队主动挂牌或合约到期，球员才进入转会市场
- 不允许强制触发其他球队球员转会

## 24. 账号系统

### 24.1 注册/登录方式

| 方式 | 说明 | 优先级 |
|------|------|--------|
| **微信登录** | 一键授权登录，自动获取微信昵称和头像 | P0（首选） |
| **手机号登录** | 短信验证码登录，需绑定手机号 | P0（必选） |

**登录流程设计：**

1. **首次登录**：
   - 微信授权 → 自动创建账号 → 绑定手机号（可选）
   - 手机号验证码 → 创建账号 → 可选绑定微信

2. **后续登录**：
   - 微信一键登录（已绑定微信的账号）
   - 手机号验证码登录

### 24.2 微信登录

**登录流程：**
1. 用户点击"微信登录"按钮
2. 调用微信SDK授权
3. 获取用户openid和unionid
4. 系统自动创建/匹配账号
5. 登录成功，进入游戏

**获取信息：**
- 微信昵称（作为默认俱乐部名称参考）
- 微信头像（作为默认头像）
- openid/unionid（用于账号识别）

### 24.3 手机号登录

**登录流程：**
1. 用户输入手机号
2. 点击"获取验证码"
3. 系统发送6位数字验证码（有效期5分钟）
4. 用户输入验证码
5. 验证通过后登录/注册

**验证码规则：**
- 长度：6位数字
- 有效期：5分钟
- 发送频率：同一手机号60秒内只能发送1次
- 每日上限：同一手机号每日最多发送10次

### 24.4 账号绑定

**绑定关系：**
- 一个账号可以同时绑定微信和手机号
- 微信和手机号可以互相绑定/解绑
- 已绑定的微信/手机号不能绑定到其他账号

**安全验证：**
- 绑定/解绑操作需要验证码确认
- 更换绑定设备需要二次验证

### 24.5 账号安全

- 登录设备管理（查看/踢出其他设备）
- 异常登录提醒（新设备登录通知）
- 账号冻结/解冻功能

### 24.6 数据安全

- 云端存档，数据不丢失
- 多设备同步（同一账号可在不同手机登录）
- 每日自动备份

### 24.7 账号限制

- 一个账号只能有一个俱乐部
- 不允许多开账号（防止作弊）
- 同一微信/手机号只能绑定一个游戏账号

## 26. 合约系统

### 26.1 合约类型

| 类型 | 适用 | 薪资 |
|------|------|------|
| 青训合约 | 18岁以下球员 | 较低（固定低薪） |
| 成年合约 | 18岁及以上球员 | 按薪资计算公式 |

- 升入一线队不影响合约类型
- 球员满18岁后，下次续约自动转为成年合约

### 26.2 合约期限

- 最短：1年
- 最长：5年

### 26.3 续约机制

- 任何时候都可以发起续约谈判
- 球员可以接受或拒绝续约
- 球员在薪资未能达到要求时，可以拒绝续约
- 薪资要求参考薪资计算公式（见第17章）

### 26.4 解约机制

- 玩家主动解约需要支付违约金
- 违约金 = 剩余合约薪资总额 × 一定比例
- 合约到期：球员自动离队，自动进入自由转会市场

### 26.5 提醒机制

- 赛季最后两周提醒
- 重点提示：合约只剩1年的球员
- 重点提示：合约即将到期的球员

### 26.6 自由市场

- 合约到期未续约的球员自动进入自由市场
- 自由市场球员无需支付转会费，只需谈薪资

## 27. 训练系统

### 27.1 训练类型

| 训练类型 | 刷新时间 | 次数 | 效果 |
|----------|----------|------|------|
| 免费训练 | 北京时间0点 | 每天1次 | 整体数值提升 +0.01×系数 |
| FC训练 | 北京时间0点 | 每天1次 | 单项属性 +0.2×系数 |
| 游戏币训练 | 随时 | 每赛季20次 | 单项属性 +0.5×系数 |

### 27.2 成长系数

综合系数 = 年龄系数 × 位置系数 × 天赋系数 × 属性差距系数

**年龄系数：**
| 年龄段 | 系数 |
|--------|------|
| 15-18 | 1.5 |
| 19-22 | 1.3 |
| 23-26 | 1.1 |
| 27-30 | 1.0 |
| 31-33 | 0.8 |
| 34-36 | 0.5 |
| 37+ | 0.2 |

**位置系数：**
- 训练位置 = 球员擅长位置：1.2
- 训练位置 ≠ 球员擅长位置：1.0

**天赋系数：**
| 潜力 | 系数 |
|------|------|
| 极高 | 1.5 |
| 高 | 1.3 |
| 较高 | 1.1 |
| 普通 | 0.9 |

**属性差距系数：**
- 属性差距系数 = 1 - 当前属性 / 天赋上限
- 属性越接近天赋上限，成长越慢
- 达到天赋上限时，成长系数为0

### 27.3 训练费用

| 训练类型 | 费用 |
|----------|------|
| 免费训练 | 0 FC |
| FC训练 | 10-50万FC（根据球员评分） |
| 游戏币训练 | 15游戏币 |

## 28. 伤病系统

### 28.1 伤病触发

**触发比赛：**
- 联赛
- 国家级杯赛（中国杯）
- 联赛杯
- 其他比赛（友谊赛、自建杯赛、洲际杯）不会受伤

**伤病概率公式：**
受伤概率 = 疲劳度基础概率 × (1 - 体质系数) × 年龄系数

**疲劳度基础概率：**
| 疲劳度范围 | 基础概率 |
|-----------|---------|
| 0-10 | 1% |
| 10-20 | 5% |
| 20-50 | 20% |
| 50-100 | 50% |

**体质系数：**
| 体质范围 | 系数 |
|----------|------|
| 1-50 | 0.3-0.5 |
| 51-75 | 0.5-0.7 |
| 76-100 | 0.7-0.9 |

**年龄系数：**
| 年龄 | 系数 |
|------|------|
| 15-25 | 0.8 |
| 26-30 | 1.0 |
| 31-35 | 1.3 |
| 36+ | 1.6 |

### 28.2 伤病恢复时间

- 1天：70%
- 3-7天：29%
- 7-30天：1%

### 28.3 恢复机制

- 自然恢复：每天自动恢复1天
- 快速恢复（游戏币）：费用 = 恢复天数 × 5游戏币
- 初愈状态：恢复后2-3天内，球员状态下降5-10%

### 28.4 伤病影响

- 受伤球员无法参加比赛
- 30岁以上球员受伤后，身体属性永久下降1-2点

### 28.5 疲劳度系统

**疲劳度来源：**
| 来源 | 疲劳度增加 |
|------|-----------|
| 参加比赛 | +10 |
| 免费训练 | +3 |
| FC训练 | +5 |
| 游戏币训练 | +5 |

**疲劳度恢复：**
- 每天自动恢复 5-10点
- 体能教练可加速恢复 +5点/天

**疲劳度上限：**
- 最高100点
- 疲劳度 > 50 → 受伤概率大幅增加

## 30. 技术架构

### 30.1 整体架构

客户端（App）→ API网关/负载均衡 → 游戏服务器（战斗服、管理服、社交服）→ 数据库集群（MySQL、Redis、MongoDB）

### 30.2 技术选型

**后端：**
- 编程语言：Go 或 Java
- 框架：Gin / Spring Boot
- 实时通信：WebSocket
- 任务队列：Redis / RabbitMQ

**数据库：**
- 主数据库：MySQL
- 缓存：Redis
- 日志/分析：MongoDB

**客户端：**
- iOS：Swift / SwiftUI
- Android：Kotlin / Jetpack Compose
- 跨平台：Flutter / React Native

### 30.3 核心系统模块

- 比赛引擎服务器：纯计算密集型，无状态，可横向扩展
- 玩家管理服务器：登录、注册、球员管理、训练、青训
- 联赛管理服务器：赛程生成、升降级计算、杯赛管理
- 社交服务器：好友系统、私信、聊天
- 定时任务服务器：比赛生成、收入发放、赛季结算

### 30.4 服务器成本估算

**初期（0-1000用户）：** 700-1500元/月
**中期（0-5万用户）：** 4000-8000元/月

### 30.5 技术优化建议

**数据持久化：**
- 定期存档，防止数据丢失
- 每个比赛日结束后自动存档
- 关键操作（如转会、签约）实时存档

**服务器架构：**
- 使用"房间制"：同一联赛的玩家在同一房间
- 减少跨服通信压力
- 联赛房间内比赛结果实时同步

**离线计算：**
- 比赛引擎提前计算所有比赛结果
- 玩家上线时直接展示结果
- 减少实时计算压力，提升响应速度

### 30.6 开发团队建议

- 后端开发：2-3人
- 前端开发：2-3人
- 游戏策划：1人
- UI/UX设计：1人
- 测试：1人
- 运维：1人（可兼职）

**初期团队：5-8人**
**总预估：8-12个月**
